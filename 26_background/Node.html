Node.JS One-Pager

By: William Guo and Jeffrey Zou

  1. What is node?

    Developed by Ryan Dahl in 2009, Node is an open-source and cross platfrom JavaScript runtime environment. It can be used to generate dynamic web page content, manipulate files on a server, collect form data, and modify data in database.

  2. Core Node concepts

    • Asynchronous

        Node's non-blocking architecture allows tasks to run simultaneously resulting in fast response times. All APIs of node.js are asynchronous meaning the server never waits for a return data and it directly moves onto the next call. This is achieved through notification mechanism of Events in Node.js that gets responses from previous API calls. (For more information: check this out)

    • Event-driven Programming

        Once an event (any action that the device perfoms: fire, create, listen (for node)) is triggered or completed, a callback function is run which only occurs in response to that event. This allows Node.js to be lightweight and memory efficient.

    • Single Threaded Architecture

        Node.Js utilizes the Libuv library along with asynchronous calling to handle thousands of concurrent threads efficiently. This is all done in the same thread which make it highly scalable and popular among top companies such as NASA, Netflix, Uber, LinkedIn and more.

  3. Modules

    Similar to libraries in JavaScript, modules of node.js allows you use to a variety of functions. In node, there are 3 types of modules:

    • Core modules - built in modules in node that are available once node is installed
    • Local modules - modules that are created by the user in the same folder as the file you are running
    • Third-party modules - modules that are created by other people and require installation through the npm

    In order to utilize a module, a require function is need in the format of const var_name = require('module_name') . For any local modules, methods use the export() keyword to make functions in that file avilable to other files. Node has multiple modules for users to use but here are some of the more popular ones:
    • HTTP module, which allows node.js to transfer data over the http. It can create a web server that listens to server ports and gives a response back to clients
    • File system module, which allows you to work with files on your computer such as reading, writing, and editing them. One to to note is that all file system functions have an asynchronous and synchronous form depending on the developers requirements.
    • Console module, which like the name suggests would allow you to print statements into the console. This is great for debugging code.

  4. Npm

    NPM stands for node package manager and this is automatically installed for you upon installation of node. Npm allows you to install packages which contain all the files required to use a module. To install a package, use the command npm install package_name and this will be stored in seperate folder named "node_modules" in the main folder.

  5. Advantages and Disadvantages of Node

    Advantages Disadvantages
    • fast and scalable due to it being run on the V8 engine
    • developers can use JavaScript for both front-end and back-end development
    • Node.js runs on multiple platforms, including Windows, macOS, and Linux, making it a versatile choice for developers
    • Bad for heavy CPU intensive tasks since it's single-threaded. For instance, if a huge request or event is in the event loop, it'll slow down processing speed.
    • “Callback hell issue” where callbacks are nested within other callbacks, making it hard to understand the code.
    • Inadequately made and documented packages from NPM or just straight up incomplete ones.
